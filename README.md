一、项目简介<br>
Unity 2D/3D 休闲烹饪模拟小游戏，使用 C# + Input System + ScriptableObject 开发。玩家限时切菜、煎菜、装盘、送订单，完成菜谱得分。<br>
二、核心玩法流程<br>
开局倒计时，限时内刷出随机订单<br>
从食材柜拿原料，切菜台切割、灶台煎制（超时会烧焦）<br>
盘子装好对应食材，送至出餐台交付订单<br>
多余食材丢垃圾桶销毁<br>
三、核心技术架构<br>
数据驱动（ScriptableObject）<br>
食材、所有菜谱、音效全部做成 SO 配置，不用改代码就能调整关卡、食材、配方。<br>
继承基类复用<br>
KitchenObjectHolder：玩家、所有柜台共用，统一处理拾取、转移、销毁物品<br>
BaseCounter：所有操作台父类，统一交互接口<br>
事件解耦<br>
切菜、上菜、捡物品等动作触发事件，音效、UI、动画只监听事件，逻辑互不干扰。
单例全局管理器<br>
GameManager（控制游戏阶段）、OrderManager（订单生成校验）、SoundManager（全局音效）。<br>
四、主要模块<br>
玩家模块<br>
Input System 控制移动、交互按键；射线检测面前操作台，同步行走动画。<br>
操作台模块<br>
食材柜、切菜台、煎灶台、装盘台、垃圾桶、出餐台、盘子自动生成台，各自实现专属烹饪逻辑。<br>
UI 模块<br>
倒计时、实时订单列表、烹饪进度条、盘子食材图标展示，动态生成界面元素。<br>
音效 & 视觉
SO 统一管理所有音效；操作台配套动画、火焰粒子、3D 盘子模型展示食材。<br>
五、项目优势 <br>
采用 SO 数据驱动，策划可快速配置内容
继承 + 事件架构，代码低耦合、易拓展新灶台 / 食材<br>
完整游戏流程：输入、交互、烹饪、订单、UI、音效闭环<br>

包含进度条、动画、粒子、射线检测等常用 Unity 功能<br>
